نظرة عامة (Overview)
-------------------------------------
"أهل القرآن" هو منصة رقمية متطورة (Backend/Frontend) مبنية باستخدام إطار عمل Django ومصممة لإدارة المسابقات القرآنية بأسلوب عصري يجمع بين الأصالة والتقنية. يهدف المشروع إلى تسهيل عملية التسجيل، التحكيم، وإعلان النتائج بأعلى معايير الدقة والجمالية، مع توفير واجهات ذات تجربة مستخدم (UI/UX) متميزة.

التقنيات المستخدمة (Tech Stack)
-------------------------------------
- لغة البرمجة: Python 3
- إطار العمل: Django & Django REST Framework (DRF)
- قواعد البيانات: SQLite (في بيئة التطوير) & MySQL (في بيئة الإنتاج) باستخدام PyMySQL
- المصادقة والأمان: JWT (JSON Web Tokens) عبر مكتبة djangorestframework-simplejwt (بصلاحية 6 ساعات للـ Access Token و30 يوماً للـ Refresh Token)
- إدارة واجهات برمجة التطبيقات: Swagger & ReDoc (عبر drf_yasg) لتوليد توثيق كامل للـ APIs
- إرسال الإيميلات: API مخصص باستخدام SMTP2GO عبر كلاس `SMTP2GOClient` المتواجد في تطبيق core
- مكتبات أخرى:
  * django-cors-headers (للتعامل مع طلبات الـ Cross-Origin)
  * django-phonenumber-field (لإدارة وحفظ أرقام الهواتف والتأكد من صحتها)
  * whitenoise (لإدارة الملفات الثابتة Static Files بفعالية وسرعة)

هيكلية المشروع (Project Architecture)
-------------------------------------
يتكون المشروع من عدة تطبيقات (Apps) رئيسية لضمان التنظيم المعماري (Clean Architecture):

1. تطبيق authentication (نظام المصادقة والمستخدمين):
   - الـ Models (قواعد البيانات):
     * Users: النموذج الأساسي للمستخدمين، مخصص ليرث من AbstractUser. يستخدم الـ UUID، ويحتوي على حقل رقم الهاتف مخصص (لمصر EG حالياً) وحقل الصلاحية/الدور Role (مستخدم عادي، طالب متسابق، مسؤول، ومشرف).
     * GoogleOAuth: لتمكين ربط الحسابات وتسجيل الدخول بضغطة واحدة بواسطة حسابات جوجل.
     * PasswordResetToken: لإدارة واستخراج رموز إعادة تعيين كلمة المرور الآمنة بصلاحية مؤقتة (10 دقائق فقط).
   - الـ APIs:
     * /auth/register/ - POST: لإنشاء حساب جديد في النظام.
     * /auth/login/ - POST: لتسجيل الدخول والحصول على رموز المرور (JWT Tokens).
     * /auth/logout/ - POST: لتسجيل الخروج بشكل آمن ومسح الـ Token بالاعتماد على Blacklist.
     * مسارات إعادة تعيين الإيميل: Reset Password Request & Confirm.

2. تطبيق websiteBackend (الواجهات وصحفات الويب):
   - مسؤول عن تقديم الردود (Views) لقوالب الويب (Templates) بشكل كلاسيكي ومباشر لدمج الفونت-إند.
   - يحتوي على المسارات (Routes) التالية:
     * / - الصفحة الرئيسية (index.html).
     * /dashboard/ - لوحة تحكم الطالب والمستخدم (dashboard.html).
     * /support/ - مركز الدعم الفني الذكي (support.html).
     * /terms/ - الشروط والأحكام.
     * /privacy/ - سياسة الخصوصية.

3. تطبيق core (الخدمات والأدوات المساعدة المشتعلة كخلفية عمل):
   - يضم إعدادات مركزية (MainVariables.py).
   - يحتوي على ملف `utils.py` الذي يضم دوال وكلاسات هامة مثل:
     * SMTP2GOClient: كلاس متطور لإرسال الإيميلات بكفاءة مع دعم الـ Templates والـ Attachments وربطه بالـ API الخاصة بخدمة SMTP2GO.
     * generate_secure_token: لتوليد رموز أمان مشفرة عشوائية قوية للمهام الحساسة.
     * redirectToNext: دالة مساعدة لتوجيه الآمن للمستخدمين.

المسارات الأساسية لواجهات برمجة التطبيقات (API Endpoints & Swagger)
-------------------------------------
للاطلاع على التوثيق الفني والتفاعلي الكامل للـ APIs واختبارها، يمكن زيارة:
- /swagger/ : واجهة Swagger UI التفاعلية الممتازة للمطورين.
- /redoc/ : التوثيق التنظيمي باستخدام ReDoc.
- /dj-admin/ : لوحة تحكم مدير Django الأساسية.

ميزات إضافية في النظام (System Features)
-------------------------------------
- تخصيص مسارات البيئة (DevMode Toggle): سهولة التبديل بين بيئة العمل ورفع التغييرات (يوجد ملف إعدادات ذكي للتبديل بين SQLite لبيئة التطوير و MySQL للإنتاج).
- Middlewares: مثل `RedirectAuthenticatedUserMiddleware` للتحكم في مسارات المستخدمين المسجلين مسبقاً وتوجيههم.
- Caching: مفعل جزئياً بناءً على الذاكرة المحلية (LocMemCache).
- User Agents & Phone Numbers: استخدام الحزم لمعالجة الأرقام والـ User-agents وتحديد الأجهزة المستخدمة.

طريقة تشغيل المشروع محلياً (How to Run Locally)
-------------------------------------
1. قم بإنشاء البيئة الوهمية (Virtual Environment) وتفعيلها في المجلد الأساسي:
   python -m venv venv
   source venv/bin/activate       # في أنظمة ماك/لينكس
   venv\Scripts\activate          # في أنظمة ويندوز

2. تثبيت الحزم والمتطلبات المدرجة في الملف:
   pip install -r requirements.txt

3. تحديث قواعد البيانات وتشغيل الـ Migrations لأول مرة:
   python manage.py makemigrations
   python manage.py migrate

4. إنشاء مستخدم سوبر أدمن (مدير النظام) للدخول للوحة التحكم:
   python manage.py createsuperuser

5. تشغيل خادم التطوير:
   python manage.py runserver

بمجرد عمل الخادم، يمكنك الوصول للمنصة عبر الروابط المذكورة بالأعلى مثل: http://127.0.0.1:8000/
